# Local Dev Tooling Options

Use project/tooling already present first.

Potential options to evaluate per project:
- Local WP or other local WordPress GUI environments.
- Docker/docker-compose if the project already uses containers.
- WordPress Playground for quick block-theme/editor experiments.
- WP-CLI for repeatable local commands and scaffolding.
- npm/Vite build pipeline when the theme already uses modern assets.
- Composer only when the theme/project already uses PHP package management or a framework like Sage.

Do not install local dev tools globally without approval.
