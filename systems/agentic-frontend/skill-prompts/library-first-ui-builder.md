# Library-First UI Builder

Use when coding frontend UI, building a page/section/component, or when the user wants high-quality implementation.

Rules:
- Library-first, custom-composed.
- Use primitives -> wrappers -> sections -> pages.
- Use Figma/source design if available.
- Use shadcn/Radix for primitives.
- Use local wrappers for brand style.
- Use section libraries as supply, not final mismatched design.
- Normalize external blocks into project design system.
- Verify rendered result when visual.
