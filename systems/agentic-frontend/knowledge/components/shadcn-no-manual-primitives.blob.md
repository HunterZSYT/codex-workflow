# Knowledge Blob: shadcn No Manual Primitives

Blob ID: shadcn-no-manual-primitives

Owner system: agentic-frontend

Owner skill: component-supply-router

Capability: Use shadcn/Radix primitives and registry components instead of hand-rolled common UI primitives.

Trigger phrases:
- shadcn
- no manual primitives
- no primitive manually
- Radix
- component registry
- use every necessary component
- accessible component primitive

When to use:
- Use when building or replacing UI controls, navigation, menus, dialogs, sheets, cards, form controls, tooltips, tabs, accordions, popovers, or other behavior-heavy components.
- Use when the user explicitly asks for shadcn or says not to manually create primitives.

When not to use:
- Do not force shadcn for unique art-directed page sections that are layout/composition rather than reusable primitives.
- Do not install every component by default.

External libraries/tools:
- `shadcn` CLI
- shadcn MCP/registry tools when available
- Radix primitives generated by shadcn components

Required docs source:
- Context7: `/websites/ui_shadcn`
- Official docs: https://ui.shadcn.com/docs
- GitHub/npm: https://github.com/shadcn-ui/ui
- Last verified: 2026-05-28

Best-practice rules:
- Check for `components.json` before assuming shadcn is initialized.
- Use the project package runner and shadcn CLI/MCP registry, not copied snippets from memory.
- Add only needed components.
- Verify generated component files and imports after adding registry items.
- Use shadcn/Radix for common primitives instead of manually coding behavior.
- Wrap primitives in project brand components when repeated styling/composition appears.
- Preserve accessibility behavior from Radix/shadcn; do not strip required titles, labels, triggers, groups, or keyboard handling.
- Use semantic project tokens for colors, spacing, and typography.
- Do not override primitive internals with brittle selectors unless necessary.

Implementation pattern:
- Inspect project config: `package.json`, `components.json`, aliases, Tailwind version, icon library.
- Search/add the specific shadcn components needed.
- Compose UI from generated primitives.
- Add brand wrappers only where reuse justifies them.
- Verify render, keyboard, focus, mobile, and component states.

Anti-patterns:
- Hand-rolled dialogs, dropdowns, sheets, tabs, select controls, command palettes, or tooltips when shadcn/Radix exists.
- Installing `--all` without a clear reason.
- Assuming `lucide-react` if `components.json` selects another icon library.
- Copying registry code without checking aliases and generated files.

Security/safety notes:
- Do not install/update dependencies without explicit approval.
- Do not fetch unknown community registry code without reviewing it.

Verification method:
- Confirm `components.json` and generated files exist.
- Render and test open/close, keyboard navigation, focus states, disabled states, and mobile behavior.
- Run build/typecheck/lint when available.

Generated/local artifacts:
- Generated shadcn components are project source and may be committed when approved.
- Registry/cache artifacts are not synced as workflow knowledge.
