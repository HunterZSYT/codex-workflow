# Editorial Layout Patterns

## Pattern 1: Metadata Rail + Main Statement

Structure:

- Left rail: section number, category, date, or compact label.
- Main area: large headline and controlled body copy.
- Optional media: aligned to the same grid.

Use for:

- agency case studies
- cultural pages
- editorial landing sections
- portfolio intros

Avoid:

- decorative labels that do not add information
- rail text too small to read
- rail positioning that breaks mobile order

## Pattern 2: Asymmetric Text + Media

Structure:

- Text spans 5 to 7 columns.
- Media spans 5 to 7 columns.
- One side has more visual weight; the other side balances through negative space or type scale.

Use for:

- hero sections
- project features
- service intros with real imagery

Avoid:

- equal 50/50 split when the design needs editorial tension
- cropped media that hides the subject
- decorative image use with no content value

## Pattern 3: Typographic Index

Structure:

- Repeated rows with strong labels or numbers.
- Consistent column starts.
- Minimal borders or rules.

Use for:

- services
- publication lists
- event/program indexes
- capabilities

Avoid:

- turning every row into a card
- adding icons when type can carry structure
