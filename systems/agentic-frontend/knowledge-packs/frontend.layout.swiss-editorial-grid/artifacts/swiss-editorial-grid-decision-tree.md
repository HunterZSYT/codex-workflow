# Swiss Editorial Grid Decision Tree

Use this candidate pack only when the answer is yes to at least two:

- Is the page meant to feel editorial, cultural, institutional, portfolio-like, or premium?
- Is typography the main visual driver?
- Would a visible modular grid improve clarity?
- Would restrained color and content-bearing imagery be better than decorative cards?
- Does the layout need asymmetry without losing order?

Do not use when:

- The UI is a dense dashboard, admin panel, CRM, table, or repeated operational workflow.
- The task is only a small spacing or copy fix.
- The user wants playful, illustrative, organic, brutalist, or maximal visual direction.
- The design would require copying a historic poster or source layout.

Smallest safe use:

1. Preserve existing content and section order.
2. Establish a shared container.
3. Add a 6/8/12-column grid only where it clarifies composition.
4. Place text and media asymmetrically.
5. Keep DOM/source order readable.
6. Verify mobile stacking.
