# Block Pattern Design Checklist

- [ ] Pattern has one clear purpose.
- [ ] Pattern uses the theme token system.
- [ ] Related elements are grouped by proximity/alignment.
- [ ] Pattern does not rely on proprietary copied layout.
- [ ] Pattern works in the editor and front-end.
- [ ] Pattern has responsive behavior checked.
- [ ] Pattern avoids excessive nested groups unless needed.
- [ ] Pattern preserves site container rhythm.
