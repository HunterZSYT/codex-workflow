# Editor / Front-End Parity Checklist

- [ ] Editor styles load the same core tokens.
- [ ] Front-end and editor use same font family sources.
- [ ] Content width and wide width match.
- [ ] Pattern spacing is not broken in editor canvas.
- [ ] Header/footer template parts render consistently.
- [ ] Button/link states remain readable.
- [ ] Block style variations appear as expected.
- [ ] Custom CSS does not hide editor controls or misrepresent front-end layout.
