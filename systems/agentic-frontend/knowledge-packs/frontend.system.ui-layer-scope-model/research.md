# Research

## Finding

The frontend system already has strong specialist skills, but tasks need a cross-scope routing model. A button, a card, a hero section, a page template, and a full site can all be affected by layout, spacing, typography, color, interaction, motion, responsive, accessibility, and state systems.

## Coverage Model

The correct model is not one more frontend skill. It is:

- a candidate capability pack for layer/scope orchestration
- precise blobs for weak layers
- micro-updates to owner routers
- registry/search/sufficiency support

## Layer Principle

Every frontend task should identify:

1. UI layer
2. UI scope
3. owner skill/blob/pack
4. local vs systemic change
5. verification method

## Scope Principle

Small scopes can create system-level consequences:

- Button padding can affect component primitives.
- Card grid spacing can affect section rhythm.
- Hero typography can affect page hierarchy.
- Navigation hover/focus can affect site-wide interaction language.
- Mobile header overflow can affect responsive structure and navigation system.
