# Project Manager Safety Policy

Default:
- classify first
- inspect before editing
- read-only first for unknown/risky tasks
- no destructive operation without explicit approval
- no fake verification
- no claiming success without evidence
- no broad QA for narrow tasks
- no screenshots for copy-only tasks unless layout risk exists
- no database/server modification without backend-database safety gates
- no frontend visual claim without frontend inspection when visual proof is needed

The Project Manager routes risky work to the correct specialist safety skill. It never approves destructive work by itself.
