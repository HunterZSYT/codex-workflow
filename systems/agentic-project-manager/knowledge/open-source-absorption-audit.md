# Open-Source Absorption Audit

Use this audit before turning any public repository into local Codex workflow knowledge, scripts, skills, or reusable patterns.

## Purpose

Open-source absorption means studying a repository as a source of ideas, patterns, APIs, workflows, or tooling decisions. It is not permission to clone into the workflow repo, copy raw code, install dependencies, activate a skill, or sync generated artifacts.

## Audit Checklist

- [ ] Retrieve local workflow knowledge first.
- [ ] Confirm whether this is a source reference, tool candidate, pattern source, script idea, or capability-pack source.
- [ ] Inspect official repository README/docs.
- [ ] Inspect license and attribution requirements.
- [ ] Inspect current activity, releases, issues, and maintenance status when relevant.
- [ ] Identify useful ideas without copying full source or proprietary assets.
- [ ] Map useful parts to existing Codex architecture.
- [ ] Decide whether the result should be a blob, candidate pack, doc update, script, MCP note, or no action.
- [ ] Keep generated reports local/task-scoped unless sanitized.
- [ ] Ask explicit approval before installing, cloning, activating, or syncing derived artifacts.

## Stop Conditions

Stop and report instead of implementing when:

- License is missing, unclear, restrictive, or incompatible.
- The repo requires credentials, private services, production access, or unsafe setup.
- The repo duplicates existing active local capability.
- The useful output is only vague inspiration, not a reusable pattern.
- User approval is needed for install, clone, dependency, MCP, config, or activation.

## Output

Create a repo absorption report and source absorption ledger entry before proposing any workflow change.
