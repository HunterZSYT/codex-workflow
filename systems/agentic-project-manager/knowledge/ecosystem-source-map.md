# Ecosystem Source Map

Use this map when scouting reusable tools, repositories, registries, components, MCPs, starter kits, libraries, templates, or official ecosystem solutions.

## 1. Official Docs / Sites

Use first when correctness, setup, API behavior, security, or compatibility matters.

Examples:

- WordPress Developer Docs
- WooCommerce Developer Docs
- GSAP docs
- Lenis docs
- shadcn docs
- Next.js docs
- Tailwind docs
- Supabase docs
- Stripe docs

## 2. Official GitHub Repositories

Use when evaluating implementation, examples, maintenance, or source-backed behavior.

Check:

- README
- examples
- docs
- releases
- issues and discussions
- license
- maintenance activity
- recent commits
- package manager support

## 3. GitHub Discovery

Use when the user did not provide a specific tool.

Search:

- GitHub repository search
- topics
- language filters
- stars
- pushed date
- `archived:false`
- license
- `template:true` when looking for starters/templates
- issue activity
- README keywords

GitHub Trending and awesome lists are discovery leads, not final proof.

## 4. Package Registries

Use package registries to check adoption, versions, compatibility, and safety.

Examples:

- npm
- Packagist / Composer
- WordPress plugin directory
- WordPress theme directory
- relevant official registries for the ecosystem

Check:

- current version
- maintenance
- downloads or adoption if available
- docs
- license
- peer dependencies
- security warnings if available

## 5. Component / Registry Ecosystems

Use for UI/component sourcing before custom UI generation.

Examples:

- shadcn registry
- Radix primitives
- Magic UI
- 21st.dev
- Tailwind component ecosystems
- WordPress block and pattern sources
- official UI libraries

## 6. MCP / Server Ecosystems

Use when a task needs live capability.

Check:

- official MCP docs
- repo health
- auth model
- local/global install fit
- security implications
- generated files
- secret handling

## 7. Curated Lists / Awesome Lists

Use only as leads. Do not treat them as authoritative.

## 8. Public Examples

Use only for pattern observation. Do not copy proprietary code, assets, exact layouts, or brand-specific implementation.
