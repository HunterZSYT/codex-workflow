# Research

LiteParse is a local PDF/document parser focused on fast spatial parsing. It provides bounding boxes, JSON/text output, Tesseract OCR by default, optional HTTP OCR servers, and conversion of Office/images into PDF before parsing. Its README also positions LlamaParse as the better fit for difficult production parsing, which supports using LiteParse as a local candidate rather than a universal default.

PaddleOCR is a much larger OCR/document-intelligence stack. The current docs describe base OCR install, optional dependency groups for document parsing, information extraction, translation, and document-to-Markdown, plus PP-OCRv5, PP-StructureV3, PP-ChatOCRv4, and PaddleOCR-VL. Context7 and official docs confirm PaddleOCR-VL is heavier and can require GPU resources.

Safe absorption is the decision logic and sync/trust boundary. Installs, model downloads, OCR HTTP services, and private-document OCR remain approval-gated.
