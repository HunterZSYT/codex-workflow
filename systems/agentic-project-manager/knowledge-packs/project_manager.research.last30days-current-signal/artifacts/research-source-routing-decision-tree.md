# Research Source Routing Decision Tree

1. Need exact implementation/API behavior?
   - Use official docs first.
   - Use Context7 for library/framework/SDK docs when available.
   - Use official repos/releases/package registries for source truth.

2. Need current ecosystem/community signal?
   - Use last30days for last-30-days chatter, sentiment, complaints, trends, GitHub/project activity leads, and source discovery.

3. Need general public/current source coverage?
   - Use web search and cite source links.

4. Need to absorb a repo?
   - Use repo absorption workflow.
   - last30days may add recent external chatter and pain points, but the repo remains primary.

5. Need reusable knowledge?
   - Run Knowledge Sufficiency Gate.
   - Convert verified facts into source ledger entries.
   - Keep weak findings candidate until verified.
