# Repo Absorption With last30days

Workflow:

1. Use repo absorption workflow as the primary process.
2. Inspect the official repo, license, README/docs, releases, issues, and generated artifacts.
3. Use last30days to find recent external chatter, complaints, adoption signals, and comparison context around the repo/tool.
4. Put community complaints into the risk section.
5. Put recent GitHub/project activity into the maturity signal only after verifying against the official repo.
6. Keep social/community sources as external signal, not primary source truth.
7. Final absorption follows the existing source ledger, architecture mapping, verification, and approval rules.
