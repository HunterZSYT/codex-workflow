# Headroom Learning Conflict Checklist

Before any `headroom learn` use:

- Confirm user approval.
- Confirm dry-run/review mode first.
- Identify which files it wants to write.
- Block writes to active workflow files until reviewed:
  - `AGENTS.md`
  - `CLAUDE.md`
  - `GEMINI.md`
  - skill prompts
  - knowledge registry
  - active packs/blobs
- Compare proposed lessons with existing Project Manager learning/error/user-feedback ledgers.
- Keep candidate lessons separate unless approved.
- Do not sync generated learning outputs unless sanitized and reviewed.
- Prefer a Project Manager enrichment candidate over direct AGENTS.md edits.
