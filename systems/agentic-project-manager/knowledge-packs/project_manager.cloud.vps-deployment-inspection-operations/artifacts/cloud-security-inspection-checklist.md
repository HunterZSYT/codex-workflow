# Cloud Security Inspection Checklist

Defensive inspection and hardening only.

## Checks

- SSL/HTTPS redirect.
- Security headers.
- Exposed sensitive files.
- Directory listing.
- Debug mode.
- File permissions.
- Environment file exposure.
- Backup/archive exposure.
- Open ports read-only.
- Outdated runtime warnings.
- WordPress admin exposure notes if relevant.
- Plugin/theme update risk notes.
- Database remote exposure check if detectable.
- Brute-force/security plugin notes for WordPress.

## Rules

- Do not reveal secrets.
- Do not run intrusive scans.
- Do not brute force.
- Do not run exploit tools.
- Do not bypass access controls.
- Do not include offensive instructions.
- Any active security testing requires explicit written scope and approval.
