# Cloud/VPS Final Proof Report

Every cloud/VPS/domain task should report:

- Target URL/domain.
- SSH target if used.
- Environment type.
- Mode used.
- Server findings.
- Public URL findings.
- Screenshots/reports generated.
- Files changed.
- Commands run.
- Backups made.
- Rollback plan.
- Security notes.
- Errors/logs found.
- Verification result.
- Remaining risks.
- What the user must approve next, if anything.

## Minimal Format

```markdown
Summary:
Target:
Environment:
Mode:
Files changed:
Commands run:
Backups/rollback:
Verification:
Security notes:
Remaining risks:
Next approval needed:
```
